1) Construção do DOM

O DOM é uma árvore que representa o HTML como nós (elementos, textos, atributos). O browser faz o parsing do HTML, tokeniza (quebra em tokens) e monta a árvore em memória.

Exemplo rápido

Um HTML simples vira uma estrutura em árvore (pai/filho):

Trecho com <code>: <button class="cta">Comprar</button> 

<main>
  <h2>Promoção</h2>
  <button class="cta">Comprar</button>
</main>

2) Construção do CSSOM

O CSSOM é a árvore de estilos. O browser interpreta o CSS, resolve a cascata (prioridade), herança e especificidade, e cria uma representação estruturada do estilo aplicável.

Um detalhe importante: o CSSOM pode ser mais custoso do que parece porque o browser precisa “entender” quais regras podem afetar quais elementos (inclusive por seletores complexos).

Dica de performance: prefira seletores mais diretos e evite excesso de re-estilizações. Isso reduz a chance de recalcular estilos com frequência.

3) Pipeline de Renderização

Da árvore ao pixel

Depois que o browser tem DOM e CSSOM, ele cria a Render Tree (uma árvore só com o que realmente será renderizado) e então calcula posições/tamanhos (Layout) e desenha (Paint).

“The DOM is a programming interface for HTML and XML documents.”

Etapas (lista ordenada)

  1. Construção do DOM (parsing do HTML → árvore de nós)
  2. Construção do CSSOM (parsing do CSS → árvore de regras/estilos)
  3. Render Tree (combina DOM + CSSOM, removendo o que não aparece, como display: none)
  4. Layout (calcula geometria: largura/altura/posição de cada item)
  5. Paint (desenha pixels: texto, cores, bordas, sombras, imagens)

Em engines modernas, pode existir uma fase extra chamada compositing (composição de camadas), mas aqui focamos no essencial. Se você ouvir “reflow” ou “relayout”, normalmente estão falando de repetir o Layout. 

parseHTML()  -> DOM
parseCSS()   -> CSSOM
combine()    -> RenderTree
layout()     -> geometry (x, y, width, height)
paint()      -> pixels

4) DOM vs CSSOM (tabela comparativa)

Os dois são “árvores”, mas com objetivos diferentes: o DOM representa conteúdo/estrutura e o CSSOM representa estilo/regras.

Aspecto DOM CSSOM
Origem HTML CSS
O que representa Estrutura do documento (nós e hierarquia) Regras de estilo (cascata, herança, especificidade)
Uso principal Manipulação de conteúdo via JavaScript Cálculo de estilos finais aplicados aos elementos
Impacto comum em performance Alterações no DOM podem disparar novo Layout/Paint Mudanças de estilo podem disparar recálculo + Layout/Paint
Combinação DOM + CSSOM → Render Tree
